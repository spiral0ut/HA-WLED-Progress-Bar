# Release Notes — v0.1.0

**Release date:** 2026-05-11  
**Minimum Home Assistant:** 2023.6.0  
**Minimum WLED firmware:** 0.12.0

---

## What's new

This is the first public release of **WLED Progress Bar** — a Home Assistant
custom integration that turns any WLED-connected LED strip into a physical
progress bar, thermometer, fluid-level display, or timer.

### Integration highlights

| Feature | Detail |
|---|---|
| Source entity | Any `sensor`, `input_number`, or `number` entity |
| Scaling | Configurable min/max value range |
| LED sub-range | Target a start–end slice of a longer strip |
| Progress colour | Flat RGB or linear gradient |
| Near-complete override | Switch colour at a configurable % threshold |
| Background | Off (saves power) or configurable dim glow |
| Reverse direction | Fill from either end |
| Brightness | Global WLED brightness control |
| Update interval | Configurable polling cadence (default 5 s) |
| Services | `render_now`, `set_colors`, `clear_bar`, `turn_off_background` |

### Lovelace card highlights

- Live 60-dot LED mini-preview matching current colours
- Current value and percentage display
- Filled / total LED count readout
- Service buttons: Render Now, Clear Bar, BG Off
- Full GUI card editor (no YAML required for basic config)
- Registers in Lovelace card picker

## Installation

### HACS (recommended)

1. In HACS → Integrations, click ⋮ → **Custom repositories**.
2. Add this repository URL, category **Integration**, and click **Add**.
3. Search *WLED Progress Bar*, click **Download**.
4. Restart Home Assistant.
5. Add the Lovelace resource (see README for exact path).

### Manual

Download `wled_progress_bar.zip` from this release, extract, and copy
`custom_components/wled_progress_bar/` into your HA `config/custom_components/`
directory. Restart Home Assistant.

## Known limitations in this release

- Only WLED **segment 0** is targeted.
- WLED **HTTP basic auth** is not supported.
- Forcing `fx: 0` (Solid) on every push can conflict with WLED presets.
- Gradient mode disables the near-complete colour override.
- No live HA + WLED integration test environment — logic follows the WLED JSON
  API spec but minor field-level adjustments may be needed.
